## 👩‍💻 구성원
* 신선림- PM
* 조호진- PD
* 임희원- FE,BE
  
## 💡 배경
* 문제점- 다양한 스타일의 여행자들을 고려한 여행 플래닝 서비스의 부족 
  * 방안 1- 여행 목적과 계획 스타일에 맞게 선택할 수 있는 다양한 플랜 템플릿 제공
  * 방안 2- 복잡한 여행 일정, 예약 변수를 한 번에 확인할 수 있는 플래닝 방식
* → **여행의 목적, 계획 스타일이 다른 국내여행자들이 개인에게 적합하게 이용할 수 있는 국내여행 공동 플래닝 서비스**

## 🛠️ 기능
* ▶️개발 진행된 파트
  * 회원가입 80%
  * 홈
    * 여행 일정 추가 50%
    * 여행 일정 상세보기 70%
    * 친구목록, 친구추가 80%
  * 여행 목록
    * 여행 목록 보기 70%

* ⏹️개발 진행 안 된 파트
  * 리뷰
  * 설정

## ⁉️ 프로젝트를 진행하면서 생긴 문제점 및 해결방안
* **데이터베이스 활용**
  * 일정 관리 앱이라는 특성상 데이터를 잘 활용해야 하는데, 데이터 활용에 어려움을 겪음.
  * 해결- 시간을 들여서 firebase공식 문서나 다른 사람들이 작성한 글을 살펴보며 해결. 

* **여행 작성 페이지 동작 매커니즘**
  * 구상했던 디자인과는 다르게 지도와 검색 부분에 의해 디자인에 맞게 개발이 힘든 영역들이 있어서 어려움을 느낌.
  * 해결- 디자인을 수정.
    
* **구글맵 검색 오류**
  * 검색 후 일정을 적는 도중에 계속해서 에러 발생.
  * 해결- GooglePlacesAutocomplete에서 debounce를 0 이상으로 설정.

## 💬 프로젝트 후기
* **목표 달성 여부**
  *  본래는 일정 추가를 완벽하게 끝내고 싶었으나, 디자인과는 다르게 알람이나 시간 표시등은 추가하지 못 함. 80% 정도 달성.
    
* **좋았던 점과 아쉬웠던 점**
  * 신선림(PM)
    * 👍 팀원들과의 소통이 아주 원활했고 평소 관심있던 분야로 서비스 주제를 선정하여 지속적인 흥미를 갖고 참여할 수 있었다.
    * 👎 서비스의 목표를 설정하는 과정에서 과감한 결단력을 발휘해야 했는데 그게 어려웠다. 목표를 명확하고 간결하게 설정하는 것이 중요하다고 느꼈다.
  * 조호진(PD)
    * 👍 좋은 사람, 좋은 프로젝트, 즐겁게 일하기
    * 👎 디자인과 개발 사이에 간극이 있었다. 그러나 소통을 통해서 개발 가능한 것과 아닌 것을 알 수 있었다.
  * 임희원(FE,BE)
    * 👍 모두가 각자의 일을 정말 열심히 하고, 소통이 매우 잘 된다.
    * 👎 분량이 많기 때문에 시간적 여유가 없어서 코드가 전반적으로 정리가 안 되어 있고, 세부적인 부분을 신경쓰지 못함.
